<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client key → Redis KEYS[1] | client-controlled key segment is namespaced into the Redis key |
| ARGV (now, cost, cfg) → Lua | untrusted-on-the-wire numerics enter the script |
| npm install | third-party package code enters the build |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Injection/Tampering | Lua KEYS / script body | mitigate | client key crosses ONLY as `KEYS[1]` (parametrized via defineCommand `numberOfKeys:1`), NEVER string-concatenated into the script body; the `.lua` source is a static file |
| T-02-05 | Tampering | ARGV numerics in Lua | mitigate | all ARGV are `tonumber()`'d; cost/now are already validated upstream (assertCost in the limiter) and are integers; the only float (`tokens`) round-trips losslessly via `%.17g` (no NaN/negative path reachable) |
| T-02-06 | Tampering | npm dependency supply chain | mitigate | T-02-SC: blocking human-verify checkpoint before install for the [ASSUMED] packages; no postinstall scripts (verified in 02-RESEARCH.md audit) |
| T-02-07 | Information Disclosure | returned Lua tuple | accept | only integers (allowed/remaining/resetMs/retryAfterMs) cross back; no key/state leaked |
</threat_model>
