<!-- swagger-ui-express@5.0.1 mount (from 05-RESEARCH §Pattern 1). Prefix mount, NOT a wildcard — Express-5 path-to-regexp@8 safe. -->
swagger-ui-express public API:
- `swaggerUi.serve` — array of static-asset middleware
- `swaggerUi.setup(spec)` — returns the request handler rendering the UI
- Mount: `app.use("/docs", swaggerUi.serve, swaggerUi.setup(openapiSpec))`
- Raw spec route (eases D-08): `app.get("/openapi.json", (_req, res) => res.json(openapiSpec))`

OpenAPI source format (D-06 discretion → RESEARCH recommends typed TS object):
- `import type { OpenAPIV3 } from "openapi-types"; export const openapiSpec: OpenAPIV3.Document = { … }`
- openapi-types is types-only, zero runtime — gives compile-time structural validation.

Exact header set to document (copy from src/adapters/express/headers.ts L53-62 + middleware.ts Retry-After):
- RateLimit-Policy: default;q=<limit>[;w=<windowSeconds>]
- RateLimit: default;r=<remaining>;t=<resetSeconds>
- X-RateLimit-Limit / X-RateLimit-Remaining / X-RateLimit-Reset
- Retry-After (429 only)
Response shapes (from README L40-65 / server.ts L104-114):
- GET /health → 200 { status: "ok" } (never throttled)
- GET /api/ping → 200 { pong: true }; AND 429 { error: "Too Many Requests", retryAfterMs: <ms> }

Docker constraint (Dockerfile L21 runs `npm ci --omit=dev`): swagger-ui-express MUST be in "dependencies", types in "devDependencies".

server.ts route ordering (LOAD-BEARING — see 05-PATTERNS server.ts section):
- /health registered BEFORE `app.use(rateLimit({ limiter }))` → unlimited
- /api/ping registered AFTER → rate-limited
- NEW: /docs + /openapi.json go in the UNLIMITED zone (before the rateLimit mount) so the UI's many static-asset requests are not throttled.
