<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host → container | The published app port (3000) is the only host-reachable surface |
| container → container | app ↔ redis over the internal compose network |
| image build → runtime | devDeps/build tooling must not leak into the runtime image |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Elevation of Privilege | app container runs as root | mitigate | `USER node` in the runtime stage (D4-05 / CLAUDE.md). Verified by grep in Task 1. |
| T-04-05 | Tampering | unpinned base images (supply-chain drift) | mitigate | Pinned `node:24-alpine` and `redis:7.4-alpine`; no `:latest` anywhere (verified by grep). |
| T-04-06 | Information Disclosure | Redis exposed on the host | mitigate | The redis service publishes NO host port; reachable only on the internal compose network. Only the app maps a port. |
| T-04-07 | Information Disclosure | secrets baked into image layers | accept | No secrets exist; REDIS_URL is a non-secret service address injected by compose. No env files copied (excluded by .dockerignore). |
| T-04-08 | Denial of Service | app starts before Redis is ready → first requests fail | mitigate | depends_on redis condition service_healthy + Redis healthcheck gate the app start (D4-05). |
</threat_model>
