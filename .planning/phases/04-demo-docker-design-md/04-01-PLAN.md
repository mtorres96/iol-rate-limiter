|----------|-------------|
| client → demo HTTP server | Untrusted HTTP requests reach /api/ping and /health |
| env → composition root | RL_ALGO / REDIS_URL / PORT control store and limiter construction |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Denial of Service | /health behind the limiter | mitigate | Register /health BEFORE app.use(rateLimit) so the healthcheck never consumes budget or is throttled (D4-03 / Pattern 2). Asserted in test/demo.test.ts. |
| T-04-02 | Tampering | RL_ALGO env value | mitigate | buildLimiter throws RangeError on any value outside token-bucket|sliding-window|fixed-window — fail loud at startup, no silent coercion (validate.ts convention). |
| T-04-03 | Spoofing | per-IP key via X-Forwarded-For | accept | Demo runs directly (no proxy) under compose; rateLimit uses req.ip (Phase-3 D3-02). Documented as a deployment note in plan 03's DESIGN.md/README. No XFF parsing added here. |
| T-04-SC | Tampering | npm install of tsx (dev-only) | mitigate | tsx is [ASSUMED]; Task 1 is a blocking-human checkpoint verifying it on the npm registry before install. tsx never ships in the runtime image (dependencies unchanged). |
</threat_model>
